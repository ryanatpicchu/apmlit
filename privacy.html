Privacy Policy

TL;DR — Amplit is a personal-use tool operated by a single independent developer. It accesses data from the developer's own Threads and Instagram accounts, plus public API data. It does not collect, sell, or share any third-party user data beyond what is publicly available through official Meta APIs.

1. Overview

Amplit ("the app") is a personal-use social media content automation tool operated by Ryan Hu ("the developer", "we", "us"), an independent developer based in Taiwan. This policy describes what data Amplit accesses, how that data is used, where it is stored, and how long it is retained.

Amplit is not a commercial product. It is not offered as a service to clients. It does not act as a data processor on behalf of third parties. The only users of Amplit are the developer and accounts explicitly added as app testers for development purposes.

2. What data Amplit accesses

Amplit accesses data in the following categories, all through official Meta APIs:

CategorySourcePurpose
Own account posts Threads API, Instagram Graph API Display the developer's own published content in a private dashboard.
Own account insights Threads API, Instagram Graph API Analyze engagement metrics (views, likes, replies, reposts) on the developer's own posts.
Replies on own posts Threads API Show replies received on the developer's own posts for manual review and response.
Follower counts (aggregate) Threads API, Instagram Graph API Track growth trend of the developer's own accounts over time. Aggregate numbers only — no individual follower identity.
Public keyword search results Threads keyword_search API Research trending topics to inform content direction. Results displayed on demand; not persisted.
Authentication tokens Meta OAuth flow Authorize Amplit to call Meta APIs on behalf of the developer's own accounts.

3. What data Amplit does NOT collect

  • Private messages (DMs) — Amplit does not access Threads or Instagram direct messages.
  • Follower lists — Amplit does not enumerate, store, or profile individual followers.
  • Other users' private data — Amplit only interacts with the developer's own accounts.
  • Payment information — Amplit does not process payments.
  • Device fingerprints, advertising identifiers, or tracking pixels.

4. How data is used

Data accessed via Meta APIs is used exclusively for:

  • Displaying the developer's own content and performance in a private dashboard.
  • Generating AI-assisted post drafts via Anthropic Claude. Only the developer's brand settings and selected trending topics are sent to the AI — never follower PII or reply content containing personal information.
  • Scheduling and publishing posts to the developer's own accounts.
  • Drafting suggested replies for the developer to manually review and send.

Each third-party service operates under its own privacy policy. Amplit does not share data with advertising networks, analytics platforms, or data brokers.

6. Data storage and retention

All data accessed by Amplit is stored in a private PostgreSQL database running on infrastructure controlled by the developer. Media files are stored in a Cloudflare R2 bucket with access restricted to the developer.

  • OAuth tokens: Stored encrypted, refreshed per Meta's token lifecycle. Revoked immediately upon disconnecting an account.
  • Post and insight data: Retained as long as the developer chooses to keep historical records for analysis. Deletable at any time via the app.
  • Reply data: Retained for engagement tracking. Not exported or shared externally.
  • Public search results: Not persisted — fetched on demand and discarded after display.
  • Media files: Retained until the developer deletes them from the media library.

7. Security

Amplit applies the following security measures:

  • All external API traffic uses HTTPS/TLS.
  • OAuth tokens are stored encrypted at rest.
  • Database access is restricted to the developer's infrastructure; no public network exposure.
  • Media uploads strip EXIF metadata before storage to avoid leaking location data.
  • The app runs behind authentication — no public access to the dashboard.

8. Your rights

Because Amplit is a personal-use tool with no external user base, the only data subject is the developer. However, if you are a Threads or Instagram user whose public data may appear in keyword search results viewed by the developer, you retain all rights granted by Meta's platform policies and applicable data protection regulations. You may control visibility of your content through your Threads / Instagram privacy settings.

Amplit does not retain public search results beyond the session in which they are fetched, and does not build profiles of any third-party user.

9. Compliance with Meta Platform Terms

Amplit operates in compliance with:

  • Meta Platform Terms
  • Developer Policies
  • Threads API Terms

Amplit does not replicate Meta's core user-facing functionality, does not sell data obtained from Meta APIs, and does not combine Meta API data with data from other sources for profiling or ad targeting.

This policy may be updated to reflect changes in Amplit's data handling practices or new requirements from Meta Platform Terms. The effective date at the top of this page indicates when the current version took effect. Significant changes will be noted in the app's changelog.
